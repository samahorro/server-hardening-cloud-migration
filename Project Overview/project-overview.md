# Project Overview

## Purpose

The purpose of this project was to document a repeatable process for hardening Windows Server systems and preparing them for migration into a cloud or hybrid infrastructure environment.

## Problem Statement

Many environments accumulate configuration drift over time. Servers may continue running unnecessary services, legacy protocols, overly broad firewall rules, inconsistent audit settings, or undocumented dependencies. These gaps increase operational risk and make migration harder.

## Objectives

- Establish a clearer server baseline
- Reduce unnecessary services and exposed ports
- Improve logging and audit coverage
- Review access control and administrative exposure
- Document migration dependencies
- Validate server functionality after hardening
- Prepare rollback steps before major changes

## My Contributions

- Reviewed existing server configuration and service exposure
- Documented hardening steps and validation requirements
- Created PowerShell-based hardening logic for repeatable configuration
- Mapped hardening actions to NIST SP 800-53 control families
- Built migration planning documentation for cloud or hybrid readiness
- Organized findings into a format suitable for IT operations and leadership review
